問題定義と実行サンプル

1. 問題文

英小文字のみの配列 a と、a の全文字を含み1文字だけ長い配列 b を受け取り、b にのみ格納されている文字を表示するアルゴリズム。

※配列の要素番号は 1 から始まる。

大域: 文字型の配列: c

c ← {"a", "b", "c", "d", "e", "f", "g", "h", "i", "j", "k", "l", "m", "n", "o", "p", "q", "r", "s", "t", "u", "v", "w", "x", "y", "z"}

入力サンプルの配列

配列 a
a
b
c
配列 b
e
a
b
c

アルゴリズム実行結果

"e"

アルゴリズムの概要と処理の推移

アルゴリズムの概要

2つの配列の「文字出現頻度の差分」を利用します。全ての文字のカウントを一度増やし、元々あった文字分を減らすことで、追加された1文字を抽出します。

補助関数 instr(x) の役割

文字 'a' を 1、'e' を 5 というように、文字を配列 c の添字（数値）に変換します。これにより、文字を直接配列 n のインデックスとして扱えます。

STEP 1: 1つ目のfor文実行後（nの初期化）
STEP 2: 2つ目のfor文実行後（配列bを走査 ➔ カウントアップ）
走査対象 配列 b:
e
a
b
c
STEP 3: 3つ目のfor文実行後（配列aを走査 ➔ カウントダウン）
走査対象 配列 a:
a
b
c
結果：配列 a の走査で相殺されなかった n[5] が「0でない値」として残る ➔ c[5] の "e" が正解！

アルゴリズムの特徴と論理

for ( i を 1 から b の長さまで ) k ← instr(b[ i ]) n[ k ] ← n[ k ] + 1 // 【1】カウントアップ endfor for ( i を 1 から a の長さまで ) k ← instr(a[ i ]) n[ k ] ← n[ k ] - 1 // 【2】カウントダウン endfor if ( n[ i ] が 0 でない ) c[ i ] を表示 // 【3】残った1文字 endif

「出現頻度の差分」による特定

ハッシュマップ的な発想に基づき、すべての文字の出現回数の差を計算することで答えを導き出します。

効率的な計算量 O(n)

二つの配列をそれぞれ一度ずつ走査するだけなので、文字の並び順に関わらず極めて高速に動作します。

テーブル参照の活用

補助関数 instr と大域配列 c を用いて、アルファベットを 1～26 の数値（添字）に変換して管理しています。

完成プログラム：extracharacter

大域: 文字型の配列: c ← {"a", "b", "c", "d", "e", "f", "g", "h", "i", "j", "k", "l", "m", "n", "o", "p", "q", "r", "s", "t", "u", "v", "w", "x", "y", "z"}
/* 要素番号は 1 から始まる */

○ extracharacter(文字型の配列: a , 文字型の配列: b)
整数型の配列: n
整数型: i , d , k
for ( i を 1 から 26 まで 1 ずつ増やす )
n[ i ] ← 0
endfor
for ( i を 1 から b の長さまで 1 ずつ増やす )
k ← instr(b[ i ])
n[ k ] ← n[ k ] + 1 /* 【1】bの文字個数をカウントアップ */
endfor
for ( i を 1 から a の長さまで 1 ずつ増やす )
k ← instr(a[ i ])
n[ k ] ← n[ k ] - 1 /* 【2】aに含まれる文字分をマイナス */
endfor
for ( i を 1 から 26 まで 1 ずつ増やす )
if ( n[ i ] が 0 でない )
c[ i ] を表示 /* 【3】残った1文字を表示 */
break
endif
endfor

○ 整数型: instr(文字型: x)
整数型: i
for ( i を 1 から 26 まで 1 ずつ増やす )
if ( c[ i ] が x と等しい )
return i
endif
endfor
