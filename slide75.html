16進数文字列の加算アルゴリズム

筆算と同じ論理で、文字としての数値を足し合わせる

   x = 1 a 2 f
+) y = 0 5 3 4
   z = 1 f 6 3
💡 16進数の対応表 (配列 n)
0〜9 → 数値の 0〜9
a〜f → 数値の 10〜15
  • 文字（"a"や"f"）のままでは計算できないため、一度数値に変換して計算する。
  • 私たちが筆算をする時と同じように、「右端（下位桁）」から計算を始める。
  • 各桁の和が16以上になった場合は、桁上がり（キャリー c）を次の桁へ渡す。
理論

空欄を解くための思考トレース（下位桁からの筆算）

CASE 1：繰り上がりのある桁（右端 i=3）
① f(15) と 4(4) と桁上がり c(0) を足す。
② t = 19。16を超えるため、16進数の文字と次の桁へのキャリー（c）を求める。
"3" (z[3]) 　商 1 → 次の桁上がり c=1
CASE 2：次の桁の計算（i=2）
① 2(2) と 3(3) に、下位から受け取った桁上がり c(1) を足す。
② t = 6。16未満なのでそのまま文字にする。
"6" (z[2]) 　商 0 → 次の桁上がり c=0
実装

疑似言語への当てはめ

○ addhex(文字配列: x, 文字配列: y, 文字配列: z) 整数型: i, c, t c ← 0 /* 右端(下位)から左端(上位)へループ */ for ( i を 3 から 0 まで 1 ずつ減らす ) t ← tonum(x[i]) + tonum(y[i]) + c z[ i ] ← n[ t mod 16 ] /* 【 １ 】 */ c ← t ÷ 16 /* 【 ２ 】 */ endfor ○ 整数型: tonum(文字型: a) 整数型: i for ( i を 0 から 15 まで 1 ずつ増やす ) if ( a が n[ i ] と等しい ) return i /* 文字に対応する数値を返す */ endif endfor

💡 【1】現在の桁の文字を決定

和 `t` を 16 で割った「余り (mod)」が、その桁に残る数値です。
ただの数値ではなく文字として出力配列 `z` に格納するため、大域配列 `n` のインデックスとして指定し、対応する文字を取得します。
解答：n[ t mod 16 ]

💡 【2】桁上がり（キャリー）の算出

次の桁へ送る繰り上がり分は、和 `t` の中に「16がいくつ含まれるか」で決まるため、16 で割った「商 (÷)」を求めます。
疑似言語の整数型同士の割り算は自動で小数点以下が切り捨てられ、商が求まります。
解答：t ÷ 16

💡 まとめ

このアルゴリズムの核心は、「法（16）を用いた剰余（余り）と商の計算」に分解すれば、2進数や16進数など、あらゆる進法の計算が私たちが普段行う「10進数の筆算」と全く同じ論理でプログラム化できることを示しています。
