練習75：解説（探索の終了地点）

2/4
なぜ text_len - pat_len まで調べれば良いのか？
  • 残り文字数：パターンの長さ（pat_len）よりも、テキストの残り文字数が短くなれば、絶対に一致することはない。
  • 効率化：無駄な比較を避けるため、最後のパターンがテキストの末尾にちょうど重なる位置で探索を止める。
  • 式：最後の開始位置 i は text_len - pat_len となる。

練習75：解説（内側の照合）

3/4
while文が止まる2つの理由を整理する。
  • 一致（発見）：j が pat_len に達したなら、パターンの全文字が一致したことになる。
  • 不一致：text[i+j] ≠ pat[j] となった瞬間に whileループを抜け、次の i へ進む。
  • 物理的イメージ：窓を1つずつずらし、窓の中の文字を上から順に突き合わせている。

練習75：最終解答

4/4
力まかせ法（文字列探索）完成プログラム
text_len ← テキストの長さ, pat_len ← パターンの長さ
for ( i ← 0 to text_len - pat_len )
　　j ← 0
　　while ( j ＜ pat_len and text[i + j] ＝ pat[j] )
　　　　j ← j + 1
　　endwhile
　　if ( j ＝ pat_len ) return i
endfor
return -1
