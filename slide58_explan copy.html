コムソート：櫛の歯で「亀の歩み」を解消する

【アルゴリズムの要旨】

間隔 m を徐々に狭めながら比較。最初は遠く、最後は隣り合う要素を交換する。

末尾付近にある小さな値「亀」を、大きなジャンプで一気に先頭方向へ移動させる。

実装

コムソートの疑似言語

○ comb_sort(整数型の配列: tbl , 整数型: n ) 整数型: i , wk , flg , m flg ← 1 m ← n while ( ( m が 1 より大きい ) or ( flg が 1 と等しい ) ) /* 【 1 】 */ 　　flg ← 0 　　m ← m ÷ 2 　　if ( m が 1 より小さい ) 　　　　m ← 1 　　endif 　　i ← 1 　　while ( i が ( n - m ) 以下 ) /* 【 2 】 */ 　　　　if ( tbl[ i ] が tbl[ i + m ] より大きい ) 　　　　　　wk ← tbl[ i ] 　　　　　　tbl[ i ] ← tbl[ i + m ] 　　　　　　tbl[ i + m ] ← wk 　　　　　　flg ← 1 　　　　endif 　　　　i ← i + 1 　　endwhile endwhile

【 1 】：継続条件

間隔 m が 1 より大きい間、または m=1 であっても交換が発生した（flg=1）場合は処理を継続します。

【 2 】：比較範囲の制御

比較対象のペア tbl[i] と tbl[i+m] について、右側の添字 i+m が n を超えないように制御します。
