クイックソート：「分割統治」と「挟み撃ち」

【アルゴリズムの要旨】

配列から基準値（pivot）を一つ選び、それより「小さいグループ」と「大きいグループ」に分割する操作を再帰的に繰り返します。

平均計算量が最速とされる王道のソート手法であり、左右からポインタを中央に向かって走らせる「挟み撃ち」が特徴です。

理論

partition関数（分割）の詳細ステップ追跡

★ pivot (基準値 = 40) ■ i (「40以上」を探す) ■ j (「40以下」を探す)
[1] 両端から探索開始
iは「40以上」の40で、jは「40以下」の30で停止。
[2] 1回目の交換 (Swap)
条件を満たす要素を見つけたので a[i] と a[j] を交換。
[3] ポインタ進行 (再開)
iは60(≧40)、jは10(≦40)を見つけてそれぞれ停止。
[4] 2回目の交換 (Swap)
a[3] と a[4] を交換。まだ i ＜ j なので処理を継続。
[5] ポインタの交差 (終了条件)
iとjがさらに進むと、i=4, j=3 となり交差して終了。
[6] 分割完了 (境界確定)
j=3 の位置を境界とし、左側と右側のグループに分割。
実装

クイックソートの疑似言語と解答

○ quicksort(整数型の配列: a, 整数型: l, 整数型: r) 整数型: j if ( l が r より小さい ) 　　j ← partition(a, l, r) /* 分割実行 */ 　　quicksort(a, l, j) /* 左グループを再帰 */ 　　quicksort(a, j + 1, r) /* 右グループを再帰 */ endif ○ 整数型: partition(整数型の配列: a, 整数型: l, 整数型: r) 整数型: i, j, pivot 論理型: flg ← true i ← l - 1 /* 【 １ 】 */ j ← r + 1 /* 【 ２ 】 */ pivot ← a[ l ] while ( flg が true と等しい ) 　　do 　　　　i ← i + 1 　　while ( a[ i ] が pivot より小さい ) /* 【 ３ 】 */ 　　do 　　　　j ← j - 1 　　while ( a[ j ] が pivot より大きい ) /* 【 ４ 】 */ 　　if ( i が j より小さい ) 　　　　a[ i ] と a[ j ] を交換 　　else 　　　　flg ← false 　　endif endwhile return j

【 3・4 】：比較のループ条件

左からは「基準値以上」を見つけるまで進みたいので、継続条件は「基準値より小さい」間となります。
右側も同様に、基準値より大きい間戻り続けます。
