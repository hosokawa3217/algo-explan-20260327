練習71：解説（最小値の記録）

2/4
なぜ変数 min_idx を使う必要があるのか？
  • 場所を覚える：値をその都度入れ替えるのではなく、「どこに最小値があったか」という添字を記録する。
  • 一回きりの交換：内側ループで最後まで探索し終わってから、最後に一度だけ交換を行うため効率的。

練習71：解説（比較の条件）

3/4
「もっと小さい値」を見つけるための条件式を導く。
  • 比較：今見ている要素 A[j] と、今までで見つけた最小の要素 A[min_idx] を比べる。
  • 更新：もし A[j] の方が小さければ、min_idx を j で上書きし、新たな最小値とする。

選択ソート（昇順）完成プログラム
for ( i ← 0 to n - 2 )
　　min_idx ← i
　　for ( j ← i + 1 to n - 1 )
　　　　if ( A[j] ＜ A[min_idx] )
　　　　　　min_idx ← j
　　　　endif
　　endfor
　　tmp ← A[i]
　　A[i] ← A[min_idx]
　　A[min_idx] ← tmp
endfor
