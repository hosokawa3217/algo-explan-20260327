練習74：解説（範囲の更新）

2/4
なぜ low を mid + 1 に更新するのか？
  • 判定済み：中央の A[mid] はすでに key と比較し、一致しないことがわかっている。
  • 絞り込み：key の方が大きい場合、mid 以下の範囲に key は存在しない。
  • 新たな左端：次の探索範囲の左端（low）は、mid のすぐ右隣から始めればよい。

練習74：解説（終了の判断）

3/4
探索が終了するのはどのような時か？
  • 発見：A[mid] ＝ key となった瞬間、その添字を返して終了する。
  • 未発見：範囲を絞り込みすぎて low ＞ high となった時、どこにも存在しなかったと判断する。
  • 前提条件：二分探索は、データが必ず「整列（ソート）」されている必要がある。

low ← 0, high ← n - 1
while ( low ≦ high )
　　mid ← (low + high) ÷ 2
　　if ( A[mid] ＝ key ) return mid
　　if ( A[mid] ＜ key )
　　　　low ← mid + 1
　　else
　　　　high ← mid - 1
　　endif
