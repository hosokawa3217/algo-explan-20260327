先頭移動法（自己組織化探索）の概要

探索効率の最適化

一度見つかったデータは「近い将来、再び探索される可能性が高い」という性質を利用します。探索のたびにデータを先頭（添字1）へ移動させ、他を後ろへずらすことで、よく使うデータが前方に集まり、平均探索回数が減少します。

核心

ずらし処理の論理：データ破壊を防ぐ

移動先の空きを作るために、既存の要素を「1つ後ろ」にコピーします。この際、処理の方向に注意が必要です。

❌ 前から順に処理 (2 → i)

tbl[2]=tbl[1]、次にtbl[3]=tbl[2]...

後ろから順に行うことで、各データが消える前に「退避」できる！

結論：j を i から 2 まで 1 ずつ減らす 制御が必要
実装

疑似言語プログラム解説

○ search (整数型の配列：tbl, n, data) i ← 1 /* 探索フェーズ */ while ((i ≦ n) and (tbl[i] ≠ data)) i ← i + 1 endwhile /* 移動フェーズ */ if (i ≦ n) /* 後ろから順にずらし、空きを作る */ for (j を i から 2 まで 1 ずつ減らす) tbl[j] ← tbl[j - 1] endfor /* 空いた先頭に発見したデータを格納 */ tbl[1] ← data endif

Step 1: 逐次探索

配列 tbl を先頭から順に走査し、見つかった位置 i を確定させます。見つからなかった場合は i > n となります。

Step 2: 退避とシフト

for ループの制御変数 j を i から開始します。tbl[j] ← tbl[j-1] によって、1つ前の要素が現在の位置に安全にコピーされます。

Step 3: 先頭挿入

全てのシフトが終わると、tbl[1] の旧データは tbl[2] に移動済みです。最後に tbl[1] を data で上書きして完了です。
