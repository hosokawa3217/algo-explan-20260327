先頭移動法（自己組織化探索）の概要

アルゴリズムの目的：探索の高速化

一度見つかったデータは「再び探索される可能性が高い」という性質を利用します。見つかったデータを配列の先頭へ移動させることで、頻繁に使うデータへのアクセス速度を自動的に向上させます。

理論

ずらし処理の核心：コピーの順序

❌ 前（2）から順にコピー

tbl[j] ← tbl[j-1] を 2 から増やすと...

同じ値で塗りつぶされてしまう！

✅ 後ろ（i）から順にコピー

tbl[j] ← tbl[j-1] を i から減らすと...

2
1
1
↓
2

データを壊さずに右へ移動できる

代入文が tbl[j] ← tbl[j-1] なので、添字 j は 発見位置 i から 2 まで 減らす。
実装

疑似言語プログラム

○ search (整数型の配列：tbl, n, data) i ← 1 /* 探索：dataがある位置 i を特定 */ while ((i ≦ n) and (tbl[i] ≠ data)) i ← i + 1 endwhile if (i ≦ n) /* 見つかった場合 */ /* 後ろから順にずらす */ for (j を i から 2 まで 1 ずつ減らす) tbl[j] ← tbl[j-1] endfor /* 空いた先頭に格納 */ tbl[1] ← data endif

1. 線形探索

配列の先頭から data を探します。見つかった時の添字 i が処理の起点になります。

2. ずらし処理 (i → 2)

tbl[j] ← tbl[j-1]。この1行で「1つ前を自分にコピー」しています。これを後ろから順に行うのがポイントです。

3. 先頭へのセット

最後に tbl[1] に探索データを上書きして完了。これで「4」が先頭に来た新しい配列が完成します。
