中置記法と後置記法の構造的違い

中置記法 (Infix)

(
a
+
b
)
*
c

・人間が理解しやすい形式
・演算子の優先順位を制御するために「括弧」を多用する

後置記法 (Postfix)

a
b
+
c
*

・コンピュータの処理（スタック）に最適化された形式
・並び順だけで順序が決まるため「括弧」が一切不要

アルゴリズムの目的

スタックを活用し、コンピュータ用の「後置記法」を読み解きながら、
人間用の「括弧付き中置記法」へと連鎖的に組み立て直す。

オペランドの処理（if側）

英小文字（a, bなど）が見つかると、スタックポインタ sp を進めて（+1）、その値をスタックの新しい行に積みます。

演算子の出現とPop（else側）

演算子が来ると、直近の要素を取り出します。sp を1つ減らすことで、一つ前の要素へアクセスし、括弧付きの新しい式として元の場所に上書きします。

スタックポインタ(sp)の役割

sp は常に「現在スタックに積まれている有効な式の数」を示しています。演算が行われるたびに sp が減り、複数の要素が1つの大きな括弧付きの式に集約されていく様子がわかります。

アルゴリズムの意義

この「後置 ➔ 中置」の変換は、コンパイラが計算式を解析したり、数式エディタが内部データを人間用に表示したりする際に利用される、データ構造（LIFO）の最も基礎的で重要な応用例の一つです。
