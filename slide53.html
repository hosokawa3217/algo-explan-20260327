練習５３：ハッシュ法によるデータ探索

ハッシュ値の計算と衝突時の「オープンアドレス法」を理解する

1 / 6

1. 問題の概要

配列 tbl の中から、目的の値 num を探索するアルゴリズムです。

  • 格納ルール： ハッシュ関数 データ mod 100 + 1 で求めた値を要素番号（添字 1〜100）とする。
  • 衝突の発生： 算出された場所に既に別のデータがあることを「衝突（コリジョン）」と呼ぶ。
  • 回避策： 衝突時は、空き領域が見つかるまで順次「次の位置」を探す（末尾からは先頭に戻る）。

3. 衝突発生時の「循環」処理（空欄【１】）

衝突が起きた際、単純に h ← h + 1 とすると、h が 100 の次に 101 となり配列の範囲外を参照してしまいます。

  • 解決策： h + 1 の値を再びハッシュ関数に通す。
  • これにより、100 の次は 1 に戻る「循環」の動きをスマートに実現できます。
while ( (num が tbl[h] と等しくない) and (i が n 以下) )
　　/* 【 １ 】：次の位置（循環対応）へ */
　　h ← hash(h + 1)
　　i ← i + 1
endwhile

➔ 【１】の解答： ウ　h ← hash(h + 1)

4. 無限ループ防止と終了判定（空欄【２】）

データが存在しない場合、永遠に探し続けるのを防ぐため、探索回数 i が要素数 n を超えたらループを打ち切ります。

  • ループ後の判定： ループを抜けた理由が「発見」か「未発見（上限到達）」かを切り分ける必要があります。
  • 直前の while 文の条件 i が n 以下 を満たしたままループを抜けたということは、途中でデータを発見した証拠です。
if ( num が tbl[h] と等しい )  /* もしくは i が n 以下 【 ２ 】 */
　　return h

➔ 【２】の解答： カ　i が n 以下

5. 解答まとめ

正解

  • 【 １ 】： (1) ウ
  • 【 ２ 】： (2) カ

ハッシュ法は理論上 O(1) での高速な探索が可能ですが、衝突時には線形探索に近づき効率が低下します。本プログラムの「循環の制御」や「探索回数による打ち切り」は、堅牢な検索システムを構築するための重要な基礎知識です。
