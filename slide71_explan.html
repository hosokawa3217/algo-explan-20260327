概要

練習71：全体像と「重複なし」の自然な実現

while ( j が n 以下 )
　if ( moji[ i ] が ptn[ j ] と等しくない )
　　// 不一致の処理...
　endif
　i ← i + 1
　j ← j + 1
endwhile

添字 i の直接操作

通常は開始位置を固定してオフセットを加算しますが、このアルゴリズムではmoji側の添字 i をループ内で直接進めて比較を行います。

「重複なし」の自然な実現

全て一致して内側のループを抜けたとき、i は既に「一致した部分の次」まで進んでいます（図の i=5 の位置）。そのため、特別なジャンプ処理を書かなくても、そのまま外側ループを続けるだけで自動的に「重複なし」が実現されます。

ポインタ i と j が「同時に進んでいく」のがこのアルゴリズムの最大の特徴です。
核心

練習71：不一致時の位置補正（バックトラック）

なぜ「戻る」必要があるのか？

図のように、途中で不一致になったとき、i は一緒に進んでしまっているため（現在i=4）、そのまま進めると探索漏れが発生します。次回の探索は「今回の開始位置の1つ右」（つまりi=3）から再開しなければなりません。

空欄(1) の位置補正（バックトラック）
i ← i - j + 2

計算式の導出

1. 今回進みすぎた分 (j - 1) を引いて開始位置に戻す。
2. 次の探索のために 1 を加える。
式： i - (j - 1) + 1 を整理して i - j + 2 となります。

不一致のときは「進みすぎたポインタを計算で引き戻す」。これが空欄(1)の論理です。
