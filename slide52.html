練習52：解説（処理のポイント）

3/5

画像2の解説内容を整理した。各処理の目的を理解しよう。

  • parent ← node：
    node が空（未定義）の場所へ移動する直前に、現在の位置を「親」として保存しておく。
  • tmp ← Tree(num)：
    新しく追加したいデータを持つ「箱」を準備する。
  • Root が 未定義 のとき：
    木が空っぽの状態であるため、生成した tmp をそのまま Root に設定する。

練習52：最終解答

5/5

treeinsert アルゴリズム完成形

if (node が 未定義)
　　tmp ← Tree (num )
　　if (Root が 未定義)
　　　　Root ← tmp
　　elseif (parent.data ＞ num )
　　　　parent.left ← tmp
　　else
　　　　parent.right ← tmp
　　endif
else
　　"エラー" を表示
endif

最終解答（選択肢）

【１】　ア．Root ← tmp
【２】　イ．parent.left ← tmp
【３】　ウ．parent.right ← tmp
