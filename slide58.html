昇順に整列された要素数 n の配列 A から値 num を探す。空欄を埋めよ。

○ binarysearch(整数型: num )
　　left ← 0,  right ← n - 1
　　while ( 【　　 1 　　】 )
　　　　mid ← ( left + right ) ÷ 2 （整数部）
　　　　if ( A[mid] == num )
　　　　　　return mid
　　　　elseif ( A[mid] ＞ num )
　　　　　　right ← 【　　 2 　　】
　　　　else
　　　　　　left ← 【　　 3 　　】
　　　　endif
　　endwhile
　　return -1

練習58：解説（二分探索の原理）

2/4

二分探索は、中央の値と比較して「左半分」か「右半分」かを捨てていく手法である。

  • 前提条件：
    データが必ず「昇順（または降順）」に並んでいなければならない。
  • 中央との比較：
    A[mid] が num より大きいなら、num は左側にあるはずである。
  • 探索範囲の更新：
    一回の比較ごとに、探索対象のデータ数が半分になるため、非常に高速である。

練習58：解説（ポインタの計算）

3/4

探索をいつまで続けるか、どうやって範囲を狭めるかを考えよう。

  • 継続条件（空欄1）：
    left が right を追い越さない間（left ≤ right）は、まだ調べる場所がある。
  • 右端を動かす（空欄2）：
    num が中央より左にあるなら、右端を中央のすぐ左へ持ってくる。つまり mid - 1。
  • 左端を動かす（空欄3）：
    num が中央より右にあるなら、左端を中央のすぐ右へ持ってくる。つまり mid + 1。

練習58：最終解答

4/4

二分探索（binarysearch）完成コード

○ binarysearch(整数型: num )
　　left ← 0,  right ← n - 1
　　while ( left ≦ right )
　　　　mid ← ( left + right ) ÷ 2 （整数部）
　　　　if ( A[mid] == num )
　　　　　　return mid
　　　　elseif ( A[mid] ＞ num )
　　　　　　right ← mid - 1
　　　　else
　　　　　　left ← mid + 1
　　　　endif
　　endwhile
　　return -1

最終解答（選択肢）

【１】 ア．left ≦ right 【２】 ウ．mid - 1 【３】 イ．mid + 1
