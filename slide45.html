練習45：二分探索

昇順に整列された配列 tbl から、目的の data を探す高速なアルゴリズムです。添字は1から開始します。

low ← 1, high ← n
while ((low が high 以下) 
       and (tbl[mid] が data と等しくない))
  if (tbl[mid] が data より小さい)
    low ← 【 1 】
  else
    high ← 【 2 】
  endif
  mid ← (low + high) ÷ 2
endwhile
補足：中央の要素と比較し、不要な半分を捨てることで、爆速でデータを見つけます。

練習45：アルゴリズムの要点

  • 二分割：1回の比較で探索範囲が半分（1/2, 1/4...）になります。
  • 圧倒的効率：100万件のデータも、わずか20回程度の比較で見つけ出せます。
  • 前提条件：対象のデータが「昇順」か「降順」に整列済みである必要があります。
補足：計算量は O(log n) と呼ばれる非常に優れた効率を持ちます。

練習45：解答の導出

if (tbl[mid] < data)
  low ← mid + 1
else
  high ← mid - 1
  • 中央が data より小さい ＝ data は右側にあるため、low（下限）を更新します。
  • そうでない ＝ data は左側にあるため、high（上限）を更新します。
  • 確認済みの mid は含めず、+1 や -1 をするのが定石です。
解答： (1) ウ mid + 1   (2) イ mid - 1

練習45：二分探索（条件分岐）

昇順の配列 tbl から data を探す際、中央の値 tbl[mid] との比較結果に応じた範囲更新を完成させなさい。

while ( (low ≦ high) and (tbl[mid] ≠ data) )
  if (tbl[mid] < data)
    low ← 【 1 】
  else
    high ← 【 2 】
  endif
  mid ← (low + high) ÷ 2 （整数部）
endwhile
補足：二分探索では、1回の比較で探索対象が半分になります。

練習45：探索範囲の絞り込み

  • tbl[mid] < data のとき：
    • 中央の値よりも目的のデータの方が「大きい」。
    • つまり、右半分（大きい側）にデータがある。
    • 下限 low を mid の右隣へ移動させる。
  • それ以外 のとき：
    • 中央の値よりも目的のデータの方が「小さい」。
    • つまり、左半分（小さい側）にデータがある。
    • 上限 high を mid の左隣へ移動させる。
補足：mid自身は既に比較済みなので、範囲から除外します。

練習45：解答の導出

if (tbl[mid] < data)
  low ← mid + 1
else
  high ← mid - 1
endif
  • この処理により、`low` または `high` が更新され、次回の `mid` は新しい範囲の中央を指します。
  • 「右へずらす＝最小値を増やす」
  • 「左へずらす＝最大値を減らす」
  • と覚えると直感的です。
解答： (1) mid + 1, (2) mid - 1
