練習５０：単方向リストの逆順処理

単方向リストのつながりをすべて逆にし、末尾を先頭にするアルゴリズムである。空欄(1)(2)を埋めなさい。

a ← 未定義, c ← ListHead
while ( c が 未定義でない )
　　【  1  】 // 本来の「次」を退避
　　【  2  】 // ポインタを逆に向ける
　　a ← c      // 一歩進む（処理済みへ）
　　c ← b      // 一歩進む（未処理へ）
endwhile
ListHead ← a // 新しい先頭の更新
選択肢： ア. b ← c.pointer    イ. c ← a.pointer    ウ. a.pointer ← c    エ. c.pointer ← a

練習５０：解説（ポインタの退避と更新）

ポインタの向きを「逆（左）」へ変える前に、本来の「次（右）」を保存する必要がある。

  • 1. 退避：c の次の場所を b に預ける。
    これを行わないと、次の要素へ進めなくなる。
  • 2. 反転：c のポインタを a（直前）に向ける。
  • 3. 移動：a と c を一つずつ右の要素へ進める。

最後に、元の末尾（a）を新たな ListHead とする。

練習５０：解答提示

完成した逆順処理プログラムは以下の通りである。

a ← 未定義, c ← ListHead
while ( c が 未定義でない )
　　b ← c.pointer
　　c.pointer ← a
　　a ← c
　　c ← b
endwhile
ListHead ← a

ループが終了したとき、a は元の末尾要素を指しているため、それが新しい ListHead となる。

解答： (1)　ア　b ← c.pointer　　(2)　エ　c.pointer ← a
