練習55：スタックの操作（問題）

1/4

配列 Stack と変数 top を用いてスタックを実現する。データを追加する push と、取り出す pop の処理を完成させよ。

○ push(整数型: data )
　　top ← 【　　 1 　　】
　　Stack[top] ← data

○ pop()
　　value ← Stack[top]
　　top ← 【　　 2 　　】
　　return value

練習55：解説（LIFOの仕組み）

2/4

スタックは「後から入れたものが先に出る」データ構造である。

  • 変数 top の役割：
    現在スタックの一番上（最後に追加されたデータ）が配列の何番目にあるかを指し示す。
  • push（追加）の順序：
    新しい場所を確保するために top を 1 増やしてから、そこにデータを書き込む。
  • pop（取出し）の順序：
    現在の top にあるデータを取り出した後、一つ下の位置を指すように top を 1 減らす。

練習55：解説（添字の増減）

3/4

配列の添字として top をどう変化させるべきか考えよう。

  • push：積み上げる：
    データが 1 つ増えるので、ポインタを上に進める必要がある。つまり top ← top + 1 となる。
  • pop：取り除く：
    データが 1 つ減るので、ポインタを下に戻す必要がある。つまり top ← top - 1 となる。
  • 初期状態：
    データが空のとき、top は通常 0 または -1 に設定される（実装による）。

練習55：最終解答

4/4

スタックの基本操作完成コード

○ push(整数型: data )
　　top ← top + 1
　　Stack[top] ← data

○ pop()
　　value ← Stack[top]
　　top ← top - 1
　　return value

最終解答（選択肢）

【１】 イ．top + 1 【２】 ア．top - 1
