練習77：解説（rearの移動）

2/4
データを入れた後、なぜ rear を増やすのか？
  • 次の場所を指す：rear は「次にデータを入れるべき空き場所」を常に指しておく必要がある。
  • 連続した格納：1つ入れたら、ポインタを1つ進めることで、次の enq で上書きされるのを防ぐ。

練習77：解説（先入れ先出し）

3/4
キューの最大の特徴：First-In, First-Out
  • 行列と同じ：先に並んだ人（データ）が、先にサービスを受ける（取り出される）。
  • enq と deq：入れる処理（enq）と出す処理（deq）が、配列の両端で別々に行われる。
enq したものが 先に deq される

キュー（enq処理）完成プログラム
procedure enq(data)
　　if ( rear ≧ MAX_SIZE )
　　　　print "Queue Full"
　　else
　　　　queue[rear] ← data
　　　　rear ← rear + 1
　　endif
endproc

最終解答

【１】 ア．rear ← rear + 1
