練習54：データの削除（問題）

1/4

二分探索木から指定された値を削除する処理の一部である。削除する節に子が2つある場合、右部分木の最小値で置き換える。空欄を埋めよ。

/* 子が2つの場合の処理 */
target ← node.right
parent ← node
while (target.left が 未定義でない )
　　parent ← target
　　target ← 【　　 1 　　】
endwhile
node.data ← target.data
if (parent が node と等しい )
　　parent.right ← 【　　 2 　　】
else
　　parent.left ← 【　　 3 　　】

練習54：解説（置き換えの論理）

2/4

子が2つある節を削除する場合、木の構造を維持するために「身代わり」を探す必要がある。

  • 身代わりの選定：
    「右部分木の中で最小の値」は、削除する値の次に大きい値であり、置き換えても二分探索木のルールが崩れない。
  • 最小値の探し方：
    右の子（node.right）へ移動した後、左の子がいなくなるまで（target.left が 未定義になるまで）左へ辿り続ける。
  • データのコピー：
    見つかった target.data を元の node.data に上書きすることで、実質的な削除を行う。

練習54：解説（ポインタの繋ぎ替え）

3/4

身代わりにした節（target）を、元の場所から切り離す処理を考えよう。

  • ループ内の移動（空欄1）：
    最小値を探すために、常に「左」へ進むため、target ← target.left となる。
  • 切り離しの分岐：
    身代わり節に「右の子」がいる可能性があるため、親のポインタをそこへ繋ぎ替える。
  • (1) 右の子が即、最小値だった場合（空欄2）：
    parent.right を target.right に繋ぐ。
  • (2) 左へ辿った先に最小値があった場合（空欄3）：
    parent.left を target.right に繋ぐ。

練習54：最終解答

4/4

二分探索木からの削除（子が2つの場合）完成コード

/* 子が2つの場合の処理 */
target ← node.right
parent ← node
while (target.left が 未定義でない )
　　parent ← target
　　target ← target.left
endwhile
node.data ← target.data
if (parent が node と等しい )
　　parent.right ← target.right
else
　　parent.left ← target.right
endif
