練習48：単方向リストの要素の削除

単方向リストから特定の文字 moji を削除する。削除対象を物理的に消すのではなく、ポインタを繋ぎ替えて切り離す論理である。

while ( ( cr が 未定義でない ) 
        and ( cr.data が moji と等しくない ) )
  prev ← cr  /* 直前の位置を退避 */
  cr ← cr.pointer  /* 比較位置の更新 */
endwhile
if ( cr が 未定義でない )
  if ( cr が ListHead と等しくない )
    【 1 】 ← 【 2 】
  else
    ListHead ← cr.pointer
  endif
endif

練習48：アルゴリズムの要点

  • 迂回（バイパス）：削除したい要素 cr を「飛び越えて」繋ぎ直すことで、鎖から切り離す [1]。
  • 直前要素の役割：削除のためには、cr の一つ前の要素 prev のポインタを書き換える必要がある [2]。
  • 例外処理：先頭（ListHead）を削除する場合は、前の要素がないため特別な処理が必要になる [1]。
ヒント：prev.pointer の行き先を、cr が指していた「次」へ変えます [1, 3]。

練習48：解答の導出

if ( cr が ListHead と等しくない )
  prev.pointer ← cr.pointer
  • 書き換えるべきは「直前の要素」の接続先、すなわち prev.pointer である [3]。
  • 新しい接続先は、削除対象 cr がそれまで指していた次の場所（cr.pointer）である [1]。
  • この代入により、prev から cr への道が消え、削除が完了する。
解答： (1) オ prev.pointer (2) エ cr.pointer
