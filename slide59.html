アルゴリズムの目的と役割

2/6
  • 外側ループ (i)：整列済みの境界を右へ広げる。
  • 内側ループ (j)：未整列の要素を正しい位置まで左へ運ぶ。
  • 効率性：正しい位置（自分以下の値）が見つかった瞬間に break するため、無駄な比較が少ない。

トランプを並べる感覚に近い

整列済みエリアの中に、新しいカードを「挿入」していく。

具体例：要素「3」の挿入 (i=1)

3/6
  • i=1 のとき、tbl[ 2 ] の「3」を整列対象にする。
  • 左隣の「7」と比較： 7 > 3 なので 交換。
  • 先頭に到達したため、挿入完了。

j は「1ずつ減らしながら」先頭へ向かっている。

空欄【 1 】の導出：添字 j の動き

5/6
for ( j を i から 1 まで 1 ずつ減らす )
　　if ( tbl[ j ] が tbl[ j + 1 ] より大きい )
  • 開始点：比較対象 tbl[ j + 1 ] が新要素 i + 1 番目を指す必要があるため、 j = i となる。
  • 方向：左へ運ぶため、値は 1 ずつ減らす。
  • 終了点：最も左の比較は tbl[ 1 ] と tbl[ 2 ]。よって j = 1 まで。

正解の論理：

i から 1 まで
1 ずつ減らす

隣り合う [j] と [j+1] を交換しながら、左へスライドさせる。

解答提示：挿入法

6/6
○ insertion_sort(整数型の配列: tbl ,  整数型: n )
整数型: i ,  j ,  wk
for ( i を 1 から n - 1 まで 1 ずつ増やす )
　　for ( j を i から 1 まで 1 ずつ減らす )
　　　　if ( tbl[ j ] が tbl[ j + 1 ] より大きい )
　　　　　　wk ← tbl[ j ]
　　　　　　tbl[ j ] ← tbl[ j + 1 ]
　　　　　　tbl[ j + 1 ] ← wk
　　　　else
　　　　　　break
　　　　endif
　　endfor
endfor
