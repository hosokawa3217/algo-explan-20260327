練習68：解説（ロジック）

2/4

ハッシュ関数によって計算された添字（ハッシュ値）を直接見に行く仕組み。

  • ハッシュ関数：
    「データの値」を「格納場所の番号（idx）」に変換する関数のこと。
  • 一発で探索：
    先頭から探す必要がなく、計算一回で場所がわかるため、データ量が増えても速度が変わらない。
  • シノニムの処理：
    もし計算先が既に埋まっていたら、次の空き場所を探す（オープンアドレス法）。

練習68：解説（特性）

3/4

計算量 O(1) という驚異的な速さと、配列サイズ内に収める工夫。

  • 計算量 O(1)：
    データ数 n に依存せず、常に一定の時間で探索が完了する理想的な速さ。
  • mod（剰余）の役割：
    ハッシュ値が配列のサイズを超えないよう、サイズ n で割った余りを用いる。

練習68：最終解答

4/4

ハッシュ探索法（hash_search）完成アルゴリズム

○ hash_search(target)
  idx ← hash(target)
  while ( H[idx] ≠ 0 )
    if ( H[idx] = target )
      return idx
    endif
    idx ← (idx + 1) mod n
  endwhile
  return -1

解答（選択肢）

【１】 ア．hash
【２】 エ．n
