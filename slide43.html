練習４３：ジャンプ探索（１）

1 / 5
昇順に整列された配列 tbl から、一定の幅 m で飛ばしながら目的の data を探す。 
while ( i が ( n - m + 1 ) 以下 )
　　if ( tbl[ i ] が data と等しい )
　　　　return i
　　elseif ( tbl[ i ] が data より大きい )
　　　　for ( j を 【 1 】 )
　　　　　　if ( tbl[ j ] が data と等しい )
　　　　　　　　return j
　　　　　　endif
　　　　endfor
　　　　return -1
　　endif
　　i ← i + m
endwhile
選択肢：ア i-1 から i-m+1 まで 1 増やす / イ i-1 から i-m+1 まで 1 減らす

アルゴリズムの目的と構造

2 / 5
  • 効率的な探索：線形探索より速く、二分探索よりシンプルな中間的アルゴリズムである。
  • ジャンプフェーズ：まず幅 m で大きく飛ばして data を超える地点を探す。
  • 詳細探索フェーズ：行き過ぎたことが分かったら、その「直前の区間」だけを詳しく調べる。

この「大股で進んでから少し戻る」動きがジャンプ探索の核心だ。

ヒント：dataを超えた瞬間に、答えはその「手前」にあることが確定する。

空欄【 1 】：戻って精査する範囲

3 / 5
  • 判定条件：tbl[ i ] が data より大きくなった瞬間が合図だ。
  • 調査の開始点：現在の i は data を超えているため、その一つ手前の i - 1 から調べ始める。
  • 調査の終着点：前回比較した地点の直後である i - m + 1 まで戻る必要がある。

逆方向に一つずつ調べるため、制御は「1 ずつ減らす」となる。

正解へのプロセス：超過した地点から、直前の安全な地点まで逆走する。

空欄【 2 】：末尾に残った区間

4 / 5
  • 状況：ジャンプの繰り返しで i が配列の末尾付近（n - m + 1 を超えた先）に到達した。
  • 開始点：最後のジャンプ地点の直後、つまり i - m + 1 が起点だ。
  • 終着点：本当の配列末尾である n までを漏れなく調べる。

最後は前方向に「1 ずつ増やす」線形探索で完結させる。

正解へのプロセス：最後のジャンプ地点から、配列の「本当の端」までを網羅する。

解答提示：ジャンプ探索

elseif ( tbl[ i ] が data より大きい )
　　for ( j を i - 1 から i - m + 1 まで 1 ずつ減らす )
　　　　/* 中略 */
　　endfor

endif
/* ループ終了後 */

このアルゴリズムは、磁気テープのように「戻る」操作にコストがかかるデバイスで、前進ジャンプを基本にする際に特に有効だ。

最終解答（選択肢）

【 1 】 イ．i - 1 から i - m + 1 まで 1 ずつ減らす
【 2 】 ウ．i - m + 1 から n まで 1 ずつ増やす
