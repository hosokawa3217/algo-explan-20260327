アルゴリズム解説

練習７６：余分な１文字の特定

英小文字の配列から、差分となる「1文字」を見つけ出す

1 / 5

1. 問題の概要

二つの文字型配列 a と b が与えられます。

  • 条件1： どちらも英小文字のみ格納。
  • 条件2： bの長さは、aより 1文字分長い。
  • 条件3： bはaの文字をすべて含み、さらにaにない文字を ちょうど1文字 含む。

目的： bのみに格納されている、その「余分な1文字」を表示すること。

2 / 5

2. カウンタによる収支の管理（空欄【１】【２】）

このアルゴリズムは、配列を直接突き合わせず、「個数の収支」を管理するハッシュマップ的発想に基づきます。

  • 配列bの走査： 長い方(b)の文字を数え、カウンタを増やす。
    ➔ 【１】の解答： カ n[ k ] ← n[ k ] + 1
  • 配列aの走査： 短い方(a)の文字を数え、カウンタを減らす。
    ➔ 【２】の解答： オ n[ k ] ← n[ k ] - 1

※共通する文字はプラスマイナスで相殺され、最終的に 0 になります。

3 / 5

3. 特定と表示（空欄【３】）

走査完了後、配列 n を調べます。

for ( i を 1 から 26 まで 1 ずつ増やす )
　　if ( n[ i ] が 0 でない )
　　　　c[ i ] を表示      /* 【 ３ 】：残った1文字を表示 */
　　　　break
　　endif
endfor

bにだけ含まれていた文字に対応する要素だけが 1 として残ります。その添字 i に対応する文字 c[i] を表示します。

➔ 【３】の解答： ア c[ i ]

4 / 5

4. 解答まとめ

正解

  • 【 １ 】： (1) カ
  • 【 ２ 】： (2) オ
  • 【 ３ 】： (3) ア

文字情報を配列の「添字（場所）」にマッピングし、計算量 O(n) で極めて高速に機能する合理的なアルゴリズムです。
