概要

練習73：配列 a と b の対応関係と圧縮のルール

■ 配列のデータマッピング

このアルゴリズムの注意点（圧縮ルール）

このプログラムは、文字の連続回数によって出力を切り替えます。

  • 3文字以上の連続：
    「文字」＋「連続数（数字）」に圧縮します。
    例： c,c,c,c → c, 4
  • 2文字の連続（特例）：
    圧縮せず、そのまま2文字を出力します。（圧縮しても長さが変わらないため）
    例： a,a → a, a

処理の流れ（概要）

配列 a の先頭から1文字ずつ確認し、「基準となる文字（i）」と「次に続く文字（k）」を比較します。同じ文字が続く限りカウントアップ（n）を続け、違う文字が出現した時点で、カウント数に応じた書き込みを配列 b に行います。

配列内の同一文字の連続を見つけ出し、文字数を減らす「ランレングス圧縮」の考え方です。
核心

練習73：連続カウントの詳細ステップ（cが4つ連続する場合）

登場する変数の役割

  • i : 基準となる文字の場所
  • k : 次の文字を調べる探索ポインタ
  • n : 連続している文字のカウント数
// i=4 ('c'開始) 時の処理 n ← 1 k ← i + 1 // 次から調査 while ( k ≦ m and a[ k ] = a[ i ] ) 　n ← n + 1 　k ← k + 1 endwhile

ループ終了後のジャンプ

調べ終わった文字をスキップするため、i ← k と一気にジャンプします.

STEP 1: 探索の準備

n = 1 k = 5

まず k ← i + 1 とし、a[5] と a[4] の比較を始めます.

STEP 2: 1回目の一致

n = 2 k = 6

等しいので n ← n + 1 とし、kを次(6)へ進めます.

STEP 3: 2回目の一致

n = 3 k = 7

等しいので n ← n + 1 とし、kを次(7)へ進めます.

STEP 4: ループ終了

n = 4 k = 8

a[7]で n=4 とした後、k=8 になります.a[8] は 'd' で不一致となるため、ここで while を抜けます.

whileループが終わった時点で、n には正確な連続文字数(4)が格納され、k は「次の異なる文字」の場所(8)を指しています.
解説

練習73：配列bへの書き込みと「空欄３」の正体

「空欄３」が問うていること

カウントした変数 n の値をもとに、出力用配列 b にどう書き込むかの条件分岐（if文）がポイントです.

// 1文字目は必ず書き込む b[ j ] ← a[ i ] j ← j + 1 if ( n ≧ 3 ) // 3文字以上なら数字を書き込む b[ j ] ← n j ← j + 1 elseif ( n = 2 ) // 2文字の時はそのままもう1文字 空欄 3 j ← j + 1 endif

ルール：「2文字の連続は圧縮せずそのまま出力する」

つまり、もう一度同じ文字を配列 b に入れる必要があるため、正解は b[ j ] ← a[ i ] となります.

【 n ≧ 3 の場合 】 例：'c' が4連続

【 n = 2 の場合 】 例：'a' が2連続

圧縮によってデータ長が変わらない（むしろ増えてしまう）ケースを回避するための「特例ルール」が正しく実装されているかが問われる問題です.
