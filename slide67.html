練習67：二分探索法（問題）

1/4

整列された配列から目的の値を高速に探すアルゴリズム。空欄を埋めよ。

low ← 0, high ← n - 1, pos ← -1
while ( 【　　 1 　　】 )
　　mid ← ( low + high ) / 2
　　if ( A[mid] = target )
　　　　pos ← mid; break
　　else if ( A[mid] ＜ target )
　　　　low ← 【　　 2 　　】
　　else
　　　　high ← mid - 1
　　endif
endwhile

練習67：解説（ロジック）

2/4

中央の値（mid）と比較し、探索範囲を「半分」に絞り込み続ける仕組み。

  • 中央との比較：
    A[mid]が目的の値より小さければ、それより左側には目的の値は存在しない。
  • 範囲の更新：
    次は右側の半分だけを調べればよいため、下限(low)を中央の隣(mid+1)に移動させる。
  • 繰り返し：
    これを繰り返すことで、データ数が多くても極めて短時間で探索が完了する。

練習67：解説（特性）

3/4

二分探索法を適用するための必須条件と、その圧倒的な効率性。

  • 必須条件：
    データが必ず昇順または降順に整列（ソート）されている必要がある。
  • 計算量：
    比較回数は最大でも O(log n)。データが2倍になっても、比較回数は1回しか増えない。

練習67：最終解答

4/4

二分探索法（binary_search）完成アルゴリズム

low ← 0, high ← n - 1, pos ← -1
while ( low ≦ high )
　　mid ← ( low + high ) / 2
　　if ( A[mid] = target )
　　　　pos ← mid; break
　　else if ( A[mid] ＜ target )
　　　　low ← mid + 1
　　else
　　　　high ← mid - 1
　　endif
endwhile

解答（選択肢）

【１】 イ．low ≦ high
【２】 ア．mid + 1
