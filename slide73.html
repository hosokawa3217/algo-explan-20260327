1/4
配列 A の先頭から順に目的の値 key を探すプログラムを完成させよ。
i ← 0
while ( i ＜ n and 【　　 1 　　】 )
　　i ← i + 1
endwhile

if ( i ＜ n )
　　return i  // 見つかった（添字を返す）
else
　　return -1 // 見つからなかった
endif

練習73：解説（繰り返しの条件）

2/4
while文が「次の要素へ進む」のはどんな時か？
  • 進む条件：今見ている要素 A[i] が、探している key と「一致しない」間は、次を探さなければならない。
  • 論理の反転：一致したらループを止めたいので、継続条件は「不一致（≠）」となる。
  • 番兵なし：このコードでは配列の範囲外（i < n）も同時にチェックし、安全に停止させる。

練習73：解説（結果の判断）

3/4
ループが終わった後、どうやって成功・失敗を見分けるか？
  • 成功（発見）：配列の最後まで行く前にループが止まったなら、A[i] = key になったということ。
  • 失敗（未発見）：最後まで見つからず i が n（配列のサイズ）に達してしまったなら、存在しない。
  • 境界線：i ＜ n かどうかで、発見したか否かを物理的に判定できる。

i ← 0
