ランレングス圧縮（連長圧縮）の仕組み

同一文字の連続を「文字＋数」で表現する

元データ：aaa b cccc ddd e
圧縮後　：a3 b c4 d3 e
  • 画像やテキストなどのデータサイズを減らす定番アルゴリズム。
  • 3文字以上連続する場合のみ、数字に置き換えて圧縮する。
  • 2文字以下の場合は、圧縮せずにそのまま文字を並べるルール。
  • 実装には「起点となる変数 i」と「調査を進める変数 k」を使う。
理論

空欄を解くための思考トレース

CASE 1：3文字以上連続する場合 (aaa)
① i を起点に固定。まず1文字目 "a" を出力する。
② k を i+1 から順に進めて、同じ文字を数える。
【出力結果 b】
[ a ] [ 3 ] ... 圧縮成功！
CASE 2：2文字の場合 (dd)
① 1文字目 "d" は既に出力済み。
② n=2 なので圧縮ルール対象外。そのまま出す必要がある。
【出力結果 b】
[ d ] [ d ] ... そのまま並べる
実装

疑似言語への当てはめ

while ( i ≦ m ) j ← j + 1 b[ j ] ← a[ i ] /* ★まず1文字確保 */ n ← 1 k ← i + 1 /* 【 １ 】調査開始位置 */ while ( ( k ≦ m ) and ( a[k] = a[i] ) ) n ← n + 1 /* 【 ２ 】同じなら数える */ k ← k + 1 endwhile if ( n ≧ 3 ) j ← j + 1 b[ j ] ← c[ n ] /* 圧縮：数字に置き換え */ elseif ( n = 2 ) j ← j + 1 b[ j ] ← a[ i + 1 ] /* 【 ３ 】2文字目転記 */ endif i ← k /* 次の未処理文字へ一気にジャンプ */ endwhile

💡 【1】【2】調査の開始と継続

【1】k ← i + 1
$k$ は起点 $i$ の「次の文字」から調査を始めます。

💡 【3】2文字の場合の例外処理

圧縮ルールから外れるため、文字を2つ並べます。
1文字目（a[i]）は既に転記されているため、2文字目である a[i+1] を手動で転記します。
