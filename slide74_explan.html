再帰による順列の生成（バックトラッキング）

【アルゴリズムの要旨】

配列内の n 個の要素の「すべての並び順（順列：n! 通り）」を生成して出力します。

ある位置（l）の文字を固定し、残りの部分の並べ替えを再帰で処理します。処理後は元の配置に戻す（バックトラック）ことで、すべての組み合わせを漏れなく網羅します。

初期状態： n = 3, c = [a, b, c]

【出力 5, 6】先頭が c
c b a
c a b
理論

全出力追跡 前半：出力 1/6 ～ 4/6 まで

■ l (固定位置) ■ 交換先 (i) ■ 出力確定 ■ バックトラック(戻す)
[1] 初期：先頭aを固定
l=1, i=1。同じ位置で交換(変化なし)
[2] 出力 1/6 (l=3到達)
l=2(b固定)を経て、l=3で配列を出力。
[3] 交換：末尾入替
戻って l=2, i=3。c[2]とc[3]を交換。
[4] 出力 2/6 (l=3到達)
入れ替えた状態で l=3 に到達し出力。
[5] バックトラック (戻す)
c[2]とc[3]の交換を戻す。先頭aの処理完了。
[6] 交換：先頭を b に
l=1に戻り、次は i=2。c[1]とc[2]を交換。
[7] 出力 3/6 (l=3到達)
そのまま l=3 まで進み、配列を出力。
[8] 交換：末尾入替
戻って l=2, i=3。c[2]とc[3]を交換。
[9] 出力 4/6 (l=3到達)
入れ替えた状態で l=3 に到達し出力。
理論

全出力追跡 後半：出力 5/6 ～ 完了まで

■ l (固定位置) ■ 交換先 (i) ■ 出力確定 ■ バックトラック(戻す)
[10] バックトラック (戻す)
先頭bの処理終了。c[1]とc[2]の交換を戻す。
[11] 交換：先頭を c に
l=1, 今度は i=3。c[1]とc[3]を交換。
[12] 出力 5/6 (l=3到達)
そのまま l=3 まで進み、配列を出力。
[13] 交換：末尾入替
戻って l=2, i=3。c[2]とc[3]を交換。
[14] 出力 6/6 (l=3到達)
最後の組み合わせを l=3 で出力。
[15] 完了 (すべて戻す)
末尾の交換を戻し、さらに先頭cの交換も戻して終了。
実装

順列生成の疑似言語と解答

○ permutation(文字配列: c, 整数: n, 整数: l, 整数: r) 整数型: i if ( l が r と等しい ) 　　c[ 1 ] ~ c[ n ] を出力 else 　　for ( i を l から r まで 1 ずつ増やす ) 　　　　c[ i ] と c[ l ] を交換 　　　　permutation( c, n, l + 1, r ) /* 【 1 】*/ 　　　　c[ i ] と c[ l ] を交換 /* 元に戻す */ 　　endfor endif

【 1 】：次の位置の固定（再帰）

l 番目の文字を固定したら、次は l + 1 番目以降の配置を決めるため、自身を再帰呼び出しします。
引数は c, n, l + 1, r となります。
