練習66：解説（仕組みと特徴）

2/4

線形探索（リニアサーチ）は、最も基本的で汎用性の高い探索手法である。

  • 探索のルール：
    データの並び順に関係なく（未整列でも）使用できる。
  • 手順：
    添字 0 から順に、目的の値（target）と一致するかどうかを確認していく。
  • 終了のタイミング：
    目的の値が見つかった瞬間に break でループを抜けることで、無駄な比較を避ける。

練習66：解説（ループの範囲と条件）

3/4

プログラムの正確な記述（添字の範囲と要素の指定）を確認しよう。

  • 空欄1（ループの終点）：
    配列の要素数が n のとき、最後の添字は n - 1 となる。したがって 0 から n - 1 まで繰り返す。
  • 空欄2（比較対象）：
    配列 A の現在の位置（添字 i）にある値、すなわち A[i] を取り出して target と比較する。
  • 見つからない場合：
    ループが最後まで回っても if 文が一度も成立しなければ、初期値の pos = -1 が返される。

練習66：最終解答

4/4

線形探索法（linear_search）完成コード

pos ← -1
for ( i を 0 から n - 1 まで 1 ずつ増やす )
　　if ( A[i] = target )
　　　　pos ← i
　　　　break
　　endif
endfor
return pos

最終解答（選択肢）

【１】 イ．n - 1 【２】 ア．A[i]
