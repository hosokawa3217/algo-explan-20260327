文字列照合：重複なし（一括置換のアルゴリズム）

【アルゴリズムの要旨】

文字列から特定のパターン文字列を検索し、出現回数を数えます。ただし「一度一致した部分は除外」します。

一致した場合はパターンの長さ分探索位置をジャンプさせ、不一致の場合は1文字ずつ進めるのが特徴です。

i ← i + n (3文字ジャンプ)

実装

文字列照合の疑似言語と解答

○ strings_search(文字型の配列: moji , 整数型: m , 文字型の配列: ptn , 整数型: n ) 整数型: i , j , cnt 論理型: flg cnt ← 0 i ← 1 while ( i が ( m - n + 1 ) 以下 ) 　　flg ← true 　　j ← 1 　　while ( ( j が n 以下 ) and ( flg が true と等しい ) ) 　　　　if ( moji[ i + j - 1 ] が ptn[ j ] と等しくない ) 　　　　　　flg ← false 　　　　endif 　　　　j ← j + 1 　　endwhile 　　if ( flg が true と等しい ) 　　　　cnt ← cnt + 1 　　　　i ← i + n /* 【 １ 】一致：長さ分ジャンプ */ 　　else 　　　　i ← i + 1 /* 【 ２ 】不一致：1文字進む */ 　　endif endwhile cnt を表示

【 1 】：一致した場合の分岐

パターンが完全一致（flg = true）した場合、一致した部分の直後から探索を再開するため、現在の位置 i にパターンの長さ n を加算します。
i ← i + n

【 2 】：不一致の場合の分岐

一文字でも不一致（flg = false）があった場合は、通常通り一つ右の文字から地道に調査し直すため、i を 1 だけ増やします。
i ← i + 1

💡 添字の定石表現

moji[ i + j - 1 ] は、「i 番目から数えて j 文字目」を指し示すための基本テクニックです。
