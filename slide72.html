概要

練習72：フォーマット変換と変数 last の役割

変数 last の役割

出力先である配列 b の「現在の末尾位置（どこまで文字を格納したか）」を記憶するポインタです。文字を1つ追加するたびに必ず + 1 されます。

ポインタ管理の分離

1文字が複数文字に変換される（tab → ΔΔ）ため、入力元の添字 i と、出力先の添字 last は次第にズレていきます。
そのため、i の進行とは完全に切り離して last の位置を独立して更新し続ける必要があります。

変換によって出力される文字数が変わるため、「書き込み用ポインタ（last）」が必要になります。
核心

練習72：forループによる空白挿入のステップ図式

空欄の制御文
for ( j を 1 から n まで
1 ずつ増やす )
// forループ内の処理
last ← last + 1
b[ last ] ← "Δ"

内側ループ（for文）の挙動

外側のループ（i）を「tab」の場所で一時停止させたまま、「1進めては Δ を書く」という動作を n 回繰り返します。
これにより、配列 b の末尾を正しく更新しながら連続して空白を挿入することができます。

「n 回ループを回す」＝「ポインタを進める処理と代入処理を n 回繰り返す」構造です。
